Comparing Mocha and Jest

Mocha

+ устанавливается в 1 команду
+ удобный синтаксис для описания тестов
+ не требует много зависимостей
+ не нужны никакие конфиги
+ assert
- пытается запустить всё до чего дотягивается....
- надо прописывапть фильтр по запуску тестов mocha "test/**/*-mocha.test.js"
- надо chai для assert
- надо продать душу, чтобы сослаться на исходники в тестах
- error: Cannot find module '../src/components/trends/Trends'
- До сих пор нет решения проблемы
- Думаю и другие минусы Jest вытекают

Jest

Пока устанавливаешь узнаёшь о существовании jest.config.js.
+ есть удобный jest.config.js для мапинга тестов и настройки
+ удобный синтаксис для описания тестов
+ не требует много зависимостей
+ есть coverage
+ удобный интерфейс с отображение где пробелемы при тестах
+ больно запускать, но не так критично
+ expect and toBe
- нужен jest.config.js
- не дружит хорошо с react-scripts test из-за версии jest, которая якобы там есть
- support for the experimental syntax 'jsx' isn't currently enabled
- не запустить сразу и легко...
- нужно .babelrc
- чтобы со стилями работал надо moduleNameMapper: { "\\.(css|sass)$": "identity-obj-proxy",}
- не эмулирует localStorage...
- Не работает с Link компонентами, без Route, поэтому сложно протестировать рендер частичной страницы

Итоги

Я бы использовал Jest, так как он больше обкатан и мне знаком. Также его можно сконфигурировать тонко, нежели Mocha. Но как говориться, что то... На самом деле Jest друг человека =) выбор будет за него!
